使用说明



导入前工作


方法一(移动到编译器目录,推荐,可以使用#include <VarManage.hpp>而不是#include "VarManage.hpp"导入):


一(针对Red Panda Dev-C++、Windows):


1.按住Win和R键,输入"C:\Program Files(x86)\Dev-Cpp\MinGW32\include"(x86架构32位)或"C:\Program Files\Dev-Cpp\MinGW32\include"(x86架构64位)
2.把文件移动到里面就完成了

二(针对独立的编译器):


1.定位到编译器中的"include"文件夹(主要这里是的是字母l(L的小写)不是数字1)
2.把文件移动到里面就完成了

方法二(使用环境变量,主次,可能失效):


1.使用文件管理器打开"VarManage_folder"
2.单击"Header-Source_code"文件夹并按下"复制路径"按钮
3.按下键盘上的Win和R键,输入"sysdm.cpl"(注意这里是字母l(L的小写)不是数字1)
4.选择"高级"选项卡,左键双击"环境变量"
5.找到"系统变量"并双击"Path"接着点"新建"粘贴刚才的路径
6.一路点击"确定"直到除了"开始"以外的所有窗口关闭

方法三(使用相对路径):

1.打开"VarManage文件夹"
2.打开"Header-Source_code"文件夹并找到"VarManage.hpp"
3.右键单击"VarManage.hpp"选择"复制"
4.粘贴到需要用的同目录里


用法


1.添加变量(add)


功能：向管理列表中添加变量，记录变量名和状态(默认状态为 true,表示「存在」)。
示例：
var::add(VAR(score));       // 添加变量 "score"，状态默认为 true(存在)
var::add(VAR(name), true);  // 添加变量 "name"，显式指定状态为存在
var::add(VAR(temp), false); // 添加变量 "temp"，状态为 false(不存在)
返回值:true 表示添加成功,false 表示失败（通常因异常状态导致）。

2. 查询变量索引(get / used_get)


get:查询变量是否「存在」(状态为 true),返回其在列表中的索引；不存在则返回 var::not_found。
used_get:查询变量是否「被添加过」(忽略状态，即使状态为 false 也会返回索引)。
示例：
var::add(VAR(age), true);
var::add(VAR(temp), false);

long long idx1 = var::get(VAR(age));    // 返回 0(存在)
long long idx2 = var::get("temp");   // 返回 var::not_found(-1,状态为 false)
long long idx3 = var::used_get(VAR(temp)); // 返回 1(被添加过)

3. 判断变量状态(find / exist / used_exist)


find:判断索引是否有效(是否为 var::not_found)。
exist:直接返回变量是否「存在」(状态为 true)。
used_exist:直接返回变量是否「被添加过」(忽略状态)。
示例：
var::add(VAR(name), true);
var::add(VAR(temp), false);

bool has_name = var::exist(VAR(name));       // true(存在)
bool has_temp = var::exist(VAR(temp));       // false(状态为 false)
bool used_temp = var::used_exist(VAR(temp)); // true(被添加过)

4. 修改变量状态(sta)


功能:修改已有变量的状态(true 为存在,false 为不存在);若变量不存在,可直接添加并设置状态。
示例：
var::add(VAR(score), true);
var::sta(VAR(score), false);  // 将 "score" 状态改为不存在
var::sta(VAR(new_var), true); // 添加 "new_var" 并设为存在

5. 清除变量(clear / clear_all)


clear:彻底删除指定变量(从列表中移除,used_get 也无法查询到)。
注意：必须传入 true 才会执行删除。
clear_all:清空所有变量记录,释放内存。
注意：必须传入 true 才会执行。
示例：
var::add(VAR(a));
var::add(VAR(b));

var::clear(VAR(a), true);   // 彻底删除 "a"
var::clear_all(true);    // 清空所有变量

6. 输出变量状态(print_get / print_used_get)


功能：打印变量的状态(存在 / 不存在),可指定终止符(默认无)。
print_get不忽略状态
print_used_get忽略状态
示例：
var::add(VAR(name), true);
var::add(VAR(temp), false);

var::print_get(VAR(name), '\n');    // 输出：变量(variable):name 状态(state):存在(exist)
var::print_used_get(VAR(temp));     // 输出：变量(variable):temp 状态(state):存在(exist) (因忽略状态)

重命名变量(rename)


功能:将已有变量重命名;若原变量不存在,则直接添加新变量。
示例：
var::add("old");
var::rename("old", "new");  // 将 "old" 重命名为 "new"
var::rename("none", "new2"); // "none" 不存在，,直接添加 "new2"

8. 异常处理(good / cross)


good:判断当前状态是否正常(无异常)。
cross:判断是否发生越界异常(修改变量时索引无效)。
示例:
if (!var::good()) {
    var::error err = var::cross(true);  // 获取具体异常信息
    printf("异常：%s\n", err.specific_error.c_str());
}

(提示:var::used_get()/var::get()在没找到时直接输出返回值会输出-1)
